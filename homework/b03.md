## Gradle

Gradle wurde installiert und geprueft mit:

gradle -v

Ein neues Gradle-Projekt wurde erstellt mit:

gradle init

Ausgewaehlt wurden:

- Application
- Java
- Groovy DSL
- JUnit Jupiter
- Java 25

Das Projekt wurde erfolgreich gestartet mit:

./gradlew :app:run

Die Ausgabe war:

Hello World!
BUILD SUCCESSFUL

## Gradle Tasks

Alle Tasks koennen angezeigt werden mit:

./gradlew tasks

Wichtige Tasks:

- `./gradlew :app:run`: startet das Programm
- `./gradlew build`: baut das Projekt und fuehrt Tests aus
- `./gradlew test`: fuehrt Tests aus

## Projektlayout

app/src/main/java -> Java-Code
app/src/test/java -> Tests
app/build.gradle.kts -> Buildskript
settings.gradle.kts -> Projektkonfiguration
gradlew -> Gradle Wrapper

## Buildskript

Das Buildskript besteht aus mehreren Teilen:

- `plugins`: definiert Plugins wie `application`
- `repositories`: gibt Quellen fuer Abhaengigkeiten an
- `dependencies`: enthaelt Bibliotheken wie JUnit
- `application`: legt die main-Klasse fest

Das `application`-Plugin stellt Tasks wie `run`, `build` und `test` bereit.

## Java-Version

Die Java-Version wurde geprueft mit:

java -version

Zusatzlich wurde im Programm ausgegeben:

System.getProperty("java.version")

Das Projekt verwendet Java 25 LTS.

## IDE

Das Projekt wurde in IntelliJ geoeffnet. Gradle-Tasks koennen ueber das Gradle-Fenster oder den Run-Button ausgefuehrt werden.

## Spotless

Das Spotless-Plugin wurde zur automatischen Formatierung des Codes eingebunden.

Konfiguration:

spotless {
    java {
        googleJavaFormat("1.28.0")
    }
}

Ausfuehrung:

./gradlew spotlessApply
./gradlew spotlessCheck

`spotlessApply` formatiert den Code automatisch.
`spotlessCheck` prueft, ob die Formatierung korrekt ist.
